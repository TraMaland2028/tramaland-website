2. TRMLの算出方法

TRML は、石油、穀物、金属、エネルギーの世界において、1,000,000ドルが実際にどれだけの価値を持つのかを示す指標です。

基準日は 2025年02月15日。
この日を起点に、指数はその100万ドルの購買力がどのように変化しているかを示します。

原材料価格が上昇すれば、係数は上昇します。
これは、同じ資源を購入するためにより多くのドルが必要になることを意味します。
ドルは現実の世界に対して弱くなります。

原材料価格が下落すれば、係数は低下します。
これは、その100万ドルでより多くを購入できることを意味します。

TRML は鏡です。
市場の期待を映すものではなく、
現実のモノを映す鏡です。
2.1 バスケット構成（バージョン v1.0）

現在のTRMLバスケットは、世界経済の基礎となる主要なコモディティを反映している。

なぜ「=F」のティッカーを使用するのか

TRMLインデックスでは、CL=F、GC=F、ZW=F などの表記を使用している。

「=F」は、市場データ取得のための技術的なティッカー形式であり、バスケット全体にわたって統一的かつ安定的に価格データを取得することを可能にする。

これは「先物トレーダーのための指数」という意味ではない。
全構成要素に対して、共通かつ比較可能な価格ソースを採用しているという意味である。

一部の資産については、すでにハイブリッド方式を採用している。
金、銀、プラチナについては、先物データだけでなくスポット価格も計算に組み込んでいる。

これにより価格の歪みを抑え、最終的な指数を実際のコモディティ・バスケットの価値変動により近づけている。

結論

「=F」はデータ形式を示すものであり、
指数の本質は、基礎的なコモディティ・バスケットの価値変動を測定することにある。

可能な場合には、より現実的な結果を得るため、スポット価格も計算に組み込んでいる。

2.2 計算原則
TRML-base サービスで使用される数式
1. 資産ごとの相対価格

r_i = P_i(t) / P_i(0)

ここで：
P_i(t) = 現在の資産価格
P_i(0) = 基準価格（ローンチ日）

2. 先物＋スポットのハイブリッド（GC, SI, PL 用）

まず以下を計算する：

r_i_fut = P_i_fut(t) / P_i_fut(0)
r_i_spot = P_i_spot(t) / P_i_spot(0)

その後、対数加重平均による混合を行う：

r_i_hyb = exp( (1 - alpha) * ln(r_i_fut)

alpha * ln(r_i_spot) )

ここで：
alpha = 0.5

3. バスケットの加重幾何平均指数

L = SUM( w_i * ln(r_i) )

一部データが欠損している場合：

L_norm = L / SUM(w_i_covered)

Index(t) = BaseIndex * exp(L_norm)

BaseIndex = 1,000,000
最小データカバレッジ：85%

4. バスケット係数

K_raw = Index(t) / BaseIndex

5. EMA7 による平滑化

K_ema7(t) = alpha_ema * K_raw(t)

(1 - alpha_ema) * K_ema7(t - 1)

alpha_ema = 2 / (7 + 1) = 0.25

6. USD の購買力

PP_raw = 1 / K_raw
PP_ema7 = 1 / K_ema7

7. 変化率（％）

Delta_K_percent = (K_raw - 1) * 100
Delta_PP_percent = (PP_raw - 1) * 100

8. JSON 出力値（コード実装ベース）

TRML_USD_raw = 1,000,000 * K_raw
TRML_USD_ema = 1,000,000 * K_ema7

9. 単純移動平均（SMA）

SMA_n = (1 / n) * SUM_{k=0 to n-1} Index(t - k)

n ∈ {7, 30}

10. 日次変動制限（アンチノイズ）

P_i(t) は以下の範囲に制限される：

[ P_ref * (1 - 0.15) , P_ref * (1 + 0.15) ]

MAX_DAILY_MOVE = 15%

2.3 購買力
係数および購買力の計算

まずバスケット係数を算出する：

K = Index / BaseIndex

ここで：
BaseIndex = 1,000,000

ドルの購買力

PP_USD = 1 / K

基準からの購買力変化（％）

Delta_PP_percent = (PP_USD - 1) * 100

例
1. もし

K = 1.10
（バスケットが10%上昇）

この場合：

PP_USD = 1 / 1.10 = 0.909

Delta_PP_percent = (0.909 - 1) * 100
= -9.1%

ドルはバスケットに対して相対的に弱くなったことを意味する。

2. もし

K = 0.90

この場合：

PP_USD = 1 / 0.90 = 1.111

Delta_PP_percent = (1.111 - 1) * 100
= +11.1%

ドルはバスケットに対して相対的に強くなったことを意味する。

K の上昇は、コモディティ・バスケット価格の上昇と、基礎資源に対する名目通貨の購買力低下を示す。

2.4 ハイブリッド価格

金、銀、プラチナについては、ハイブリッド評価方式を採用している。

50% — 先物価格
50% — スポット価格

これにより、コンタンゴやバックワーデーションなど、先物市場特有の構造による歪みを軽減している。

5. 暗号資産の成長モデルに対する立場

TRMLは、既存の暗号資産と競争する目的で生まれたものではない。

それは、暗号資産の進化を観察する中で生まれた。

ここ数年、デジタル資産は指数関数的成長への期待の対象となってきた。
BTCが1,000,000ドルに到達することを待ち続ける、終わりの見えない期待。

多くの場合、価値は利用や基盤ではなく、将来価格の予測によって測られるようになった。
しかし予測は一様であっても、現実はしばしば異なる。

同時に、膨大な計算能力、電力、物理的資源が、分散型台帳上の数値記録を検証するためのネットワーク維持に投入されている。
その構造は妥当なのか、という問いは残る。

TRMLはこれらのモデルを「正しい」または「誤り」と評価しない。
代わりに、別の方向性を提示する。

数十年後の仮説的な価格水準を待つのではなく、TRMLは現在の実物資源基盤の状態を測定することに焦点を当てている。

抽象的価値の上昇を目的とした資源消費ではなく、TRMLは既存の経済的物質構造の変化係数を公表する。

TRMLはハッシュレート競争、発行量、計算資源の希少性に基づいて構築されていない。
透明な数式と最小限のアルゴリズム計算に基づいている。

TRMLは将来価格に賭けない。
現在の動態を記録する。

6. 技術基盤：なぜPolygonなのか

TRMLはデジタル環境に存在し、その選択は重要である。

デジタル部分の実装には、低い取引コスト、高速な承認、安定したエコシステムを備えるインフラとして、Polygonネットワークが選択された。

TRMLはエネルギー集約型の計算メカニズムを必要としない。
そのロジックはマイニングではなく、測定である。

Polygonは効率性とスケーラビリティを重視したコンセンサスモデルに基づいて構築されており、以下を可能にする：

エネルギー消費の最小化

高速なトランザクション承認

低い利用コスト

ネットワーク混雑の回避

これはTRMLの思想と整合する：

過剰な計算を減らす

障壁を減らす

アクセス可能性を高める

エコシステム

Polygonは成熟したインフラを提供している：

Ethereumとの互換性

発展した開発ツール群

安定したスマートコントラクト環境

多くのウォレットおよびサービスとの統合

TRMLは独自の計算競争を創出しようとしない。
既存の実用的なデジタルインフラを活用する立場を取っている。


7. トランザクションとネットワークガス

Polygonネットワーク上の他のデジタル資産と同様に、TRMLも標準的なブロックチェーントランザクションの仕組みを利用している。

送金やスマートコントラクトとの相互作用を含むあらゆる操作には、ネットワーク手数料、いわゆる「ガス」の支払いが必要である。

Polygonネットワークでは、ガスはネイティブトークンであるPOLで支払われる。

これはブロックチェーンインフラの技術的要件であり、TRML固有の仕組みではない。

TRMLは：

独自の手数料を徴収しない

ネットワークガスの価格を管理しない

ネットワーク手数料から利益を得ない

ガス価格は、ネットワークの混雑状況およびPolygonプロトコルのパラメータによって決定される。

透明性の原則

ブロックチェーン上でのTRMLとのあらゆる相互作用は：

トランザクション手数料支払いのための最小限のPOL保有を必要とし

ユーザーが直接実行し

中央集権的な仲介者を前提としない

TRMLは既存のインフラを利用している。
ネットワーク手数料は分散型台帳への記録に対する対価であり、プロジェクト内部の収益モデルではない。
